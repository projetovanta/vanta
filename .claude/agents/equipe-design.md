## Como Funciona

1. **Entender o usuário** — Quem estamos projetando para?
2. **Diagnosticar o desafio** — Novo produto? Feature? Sistema? Processo?
3. **Definir constraints** — Brand? Acessibilidade? Technical debt?
4. **Rotear por fase** — Research → System → Visual → Implementation
5. **Quality gates** — Checklist em cada transição
